Environnement Collaboratif GITHUB

Objectif

Ameliorer nos méthode de travail en projet

Contexte

Dans le cadre de ma formation à itic paris nous avons eux un projet à réaliser en groupe.
Nous avons donc choisi de réaliser une application de gestion de ligue en java.
Nous avons donc du mettre en place un environnement collaboratif pour travailler sur le projet.

Problématique

Comment mettre en place un environnement collaboratif pour travailler sur un projet en groupe ?

Solution

  • Nous avons choisi d'utiliser github pour mettre en place notre environnement collaboratif
  • Nous avons donc forker le repository du projet sur github
  • Nous avons ensuite exporter le projet java sur Eclipse
  • Nous avons ensuite lier notre projet java à notre repository github
  • Nous avons ensuite créer des branches pour travailler sur des itérations du projet
  • Nous avons ensuite push nos branches sur github et nous avons ensuite créer des pull request pour merger nos branches sur la branche master
  • Chaque pull-request devait être revu par deux autre membre du groupe avant d'être merger sur la branche master.
  • Nous avons ensuite push notre branche master sur github à chaque itérations

Conclusion

Grace à github nous avons pu travailler en groupe sur notre projet et nous avons pu voir les avantages de travailler en groupe sur un projet notamment :

  • La possibilité de travailler sur des itérations du projet en même temps
  • La possibilité de travailler sur des fonctionnalités différentes en même temps
  • L'obligation de review le code
  • Communiquer sur les étape clé du projet
  • La possibilité de revenir à une version antérieur du projet
  • La possibilité de travailler sur le projet en local

Compétences acquises

  • Collecter, suivre et orienter des demandes
  • Évaluer les indicateurs de suivi d’un projet et analyser les écarts
  • Développer son projet professionnel
